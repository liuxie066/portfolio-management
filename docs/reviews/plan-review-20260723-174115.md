## Assumptions tested

### Architecture boundary

- Confirmed `render_receipt()` remains the business-content source of truth.
- Confirmed the proposed conversion belongs at the outbound Feishu transport
  boundary and does not require NAV or Futu state-machine changes.
- Confirmed `send_text_message()` remains available and unrelated callers are
  not forced into the new contract.

### External protocol

- Confirmed the message-create request still uses `/im/v1/messages` with
  `receive_id_type=open_id`.
- Confirmed post content must remain a JSON string containing `zh_cn.title` and
  two-dimensional paragraph nodes.
- Confirmed title removal and exact H2 conversion prevent structural hash
  markers from leaking into the visible body.

### Failure and lifecycle behavior

- Confirmed dry-run still exits before client construction.
- Confirmed missing configuration remains a receipt-local failure.
- Confirmed conversion/client/token/HTTP/API failures remain within each
  service's current best-effort exception boundary.
- Confirmed successful delivery preserves `message_id` and failed delivery does
  not rewrite the parent Futu sync or NAV job result.

### Best-practice and optimal-solution review

- Exact serialized-payload tests exercise the external protocol boundary.
- Service fakes exposing only `send_post_message()` make regressions to the old
  interface fail deterministically.
- Input validation prevents malformed H1/body structures from reaching token or
  HTTP work.
- A native post is the smallest mechanism that can provide both an independent
  title and styled section nodes; text spelling changes cannot solve the bug.

### Overengineering and overcoupling review

- No new dependency, configuration, schema, state, class hierarchy, generic
  Markdown parser, or card framework is proposed.
- One additive client method plus two call-site substitutions keeps the work
  unit atomic without tying unrelated notification paths to it.

### Validation review

- Planned tests cover the exact `sy` title, removed H1, bold H2, Unicode,
  internal blank spacing, invalid input, both service migrations, dry-run,
  failure isolation, unchanged renderer output, full suite, compile, and diff
  checks.
