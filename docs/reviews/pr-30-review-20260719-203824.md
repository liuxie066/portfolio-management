## Adversarial Failure Pass

### Financial write and idempotency boundaries

- `TradeService` validates finite positive trade/cash-flow inputs before the ledger write, serializes account writes, plans absolute target state, and returns replayed ledger rows before applying side effects.
- Buy cash insufficiency and sell holding insufficiency are checked under the account lock before the transaction record is created.
- Transaction/cash-flow repositories fail closed on idempotency lookup errors and keep the same-host check/create sequence under a process lock.
- Secondary target failures raise `PartialWriteError`; compensation durability failure is separately visible instead of being misreported as a recoverable queued task.

### Compensation recovery

- The local append-only event log uses `O_APPEND`, flush, fsync, and a log lock; Feishu is an optional mirror rather than the authoritative recovery queue.
- Retry obtains the account lock before the task lock, folds the journal, rejects unsupported legacy delta tasks, and uses compare-and-set target application.
- Holding delete/set and NAV snapshot target recovery distinguish already-applied state from state conflict, preventing forced overwrite of changed live state.

### Futu sync and installer safety

- Only classified LONG STOCK/ETF non-zero positions enter the canonical sync set; short/unknown-side, duplicate-normalized, missing-cost, and dangerous empty snapshots fail before writes.
- The position snapshot consumes `average_cost` only; `diluted_cost` and deprecated `cost_price` are not fallback sources.
- Existing stock/ETF rows can be zeroed, while unmatched legacy fund rows are not treated as canonical zero targets.
- Installer env merge reads and validates source/target files before deployment writes, preserves unrelated content, imports only the three `OM_FEISHU_BOT_*` values, and leaves the target byte-for-byte unchanged when the source is absent.

### NAV state and repair

- Persisted NAV plus failed holdings snapshot is surfaced as `partial`/`recovery_required` with a compensation task and retry command; upstream account/daily-job layers preserve that state.
- Dry-run holdings snapshots do not overwrite the formal local snapshot artifact.
- Patch preflight resolves exact live targets, rejects missing/duplicate rows before writes, validates changed rows plus their first chronological successors, and records a stable plan digest.
- Apply/resume/rollback use append-only journals, account-before-journal lock order, compare-and-set state checks, reverse rollback order, and explicit partial results.

### Pricing and valuation

- One monotonic deadline is propagated through batch, provider, retry/backoff, HTTP, and FX paths; cache-only selection occurs before fixed-price and FX/network work.
- Quote normalization rejects non-positive/non-finite price, CNY price, and FX values; foreign quotes require `cny_price`.
- Valuation uses positive finite CNY prices, limits unit-price fallback to CNY cash/MMF, emits blocking warnings for missing foreign FX or security prices, and preserves eight-decimal snapshot quantities.
- Market-session logic uses the market-local timezone, including New York DST behavior; explicit supported suffix facts are canonicalized consistently for routing and result remapping.

### Feishu consistency boundary

- Core token/data requests use configured bounded connect/read timeouts.
- Batch create/update/delete validates response cardinality and record-ID mapping and exposes only the confirmed complete prefix in structured failures.
- Repository counts/caches advance only from confirmed results; delete failures propagate before cache invalidation.
- Live schema inspection paginates all field pages and checks documented required type families in strict mode.

### Service safety boundary

- Every `_service_or_fallback` caller explicitly declares its fallback policy: four write paths disable fallback and eight read paths allow it.
- The ASGI middleware checks only `request.client.host`, accepts IPv4/IPv6 loopback by default, and ignores spoofable forwarding/Host headers.
- Explicit remote mode remains opt-in and documented as unauthenticated; `scripts/serve.py --allow-remote` sets the application environment before Uvicorn import/run and restores it afterward.
